# Security Policy

## Supported Versions

Security fixes are applied to supported releases on the default branch and the
latest tagged release line.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use GitHub Private Vulnerability Reporting:

1. Go to the Security tab of this repository.
2. Select Advisories.
3. Choose Report a vulnerability.
4. Include reproduction details, impact, and any suggested mitigation.

If GitHub advisories are unavailable, open a private maintainer contact request
and include the same details.

## What to Expect

- Initial triage target: within 5 business days.
- We will acknowledge receipt, assess severity, and coordinate remediation.
- We may ask for validation support on candidate fixes.
- We will credit reporters when appropriate and requested.
