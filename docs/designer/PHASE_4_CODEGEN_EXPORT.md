## 1. Razor Code Generator (`CodeGen/RazorCodeGenerator.cs`)

### Purpose
Converts a `DesignPage` model into clean, production-ready Razor code with the following guarantees:

- **Deterministic**: Same input ⇒ byte-identical output (testable, repeatable)
- **Human-readable**: Proper indentation, logical parameter ordering
- **Standards-compliant**: Only non-default parameters written, token-based colors only
- **Safe for export**: Generated code must pass the repo's own quality gates (token-audit, Label/AriaLabel guards)

### Key Design Decisions

#### 1. Timestamp Handling for Determinism
- Production code: **No timestamp** in the generated header
  - Comment header: `@* Generated by Agterhuis.Ui Designer v1.0 *@`
  - Document: `Document: {name}`
- Test mode: Accepts optional `testTimestamp` parameter for reproducible tests
  - Allows byte-identical comparison of two generations

#### 2. Parameter Filtering
- Only parameters **different from registry defaults** are output
- Parameters sorted **alphabetically** for stability
- Multi-line only if ≥3 parameters; otherwise single line

#### 3. Slot Rendering
- `ChildContent` slot rendered inline (no explicit `<ChildContent>` tags)
- Named slots (`Columns`, `Items`, `Header`, etc.) explicitly tagged and indented
- Sorted deterministically by slot name

#### 4. Type Handling
- `bool`: `true` or `false` (lowercased)
- Numbers: JSON representation
- Strings: Quoted, with HTML entity escaping (`"` → `&quot;`)
- Expressions: Pass-through as C# code (for phase 5: dynamic bindings)
- `RenderFragment` / `EventCallback`: Read-only in v1 (no rendering)

### Code Structure
```
RazorCodeGenerator
├── GeneratePageCode(page, document)      // Main entry point
├── GenerateNodeCode(node, lines, level)  // Recursive tree traversal
├── GenerateParameterLines(node, desc)    // Parameter formatting
├── FormatParameterValue(value, desc)     // Type-specific formatting
└── EscapeQuotes(value)                   // HTML-safe escaping
```

### Example Output
```razor
@* Generated by Agterhuis.Ui Designer v1.0
   Document: Customer Dashboard
*@

@page "/customers"
@rendermode InteractiveWebAssembly

<PageTitle>Customers</PageTitle>

<MainLayout>
    <AgtTextField @Label="Search" />
    <RadzenRow>
        <RadzenColumn Size="12">
            <RadzenDataGrid @Data="customers" />
        </RadzenColumn>
    </RadzenRow>
</MainLayout>

@code {
    // TODO: Add component logic here
}
```

## 2. Razor Formatter (`CodeGen/RazorFormatter.cs`)

### Purpose
Encapsulates indentation and formatting logic for consistent, adjustable output.

### API
```csharp
public class RazorFormatter
{
    public RazorFormatter(int indentSize = 4);
    public string Indent(string text, int level);           // Add indent
    public string FormatParameter(string name, string value); // @Name="value"
    public string FormatParameterNoValue(string name);      // @Name (for bool true)
    public string Normalize(string text);                   // Trim trailing whitespace
}
```

### Design Notes
- **Indent size**: Configurable (default 4 spaces per level)
- **Line normalization**: Strips trailing whitespace, handles mixed line endings
- **Deterministic**: No locale-dependent formatting

### Purpose
A tabbed UI component showing:
1. **Code tab**: Live-generated Razor (read-only, with "preview" badge)
2. **Model tab**: Editable JSON representation with validation

## 4. Project Exporter (`Export/ProjectExporter.cs`)

### Purpose
Generates a complete, runnable .NET project from a `DesignDocument`.

### Design Philosophy
- **Client-side only**: No server-side build, `dotnet new`, or code generation tools
- **Self-contained**: All template files embedded as resources during build
- **Round-trip ready**: Includes `design/document.json` for re-import
- **Deterministic**: Same model + theme ⇒ same project structure

### Core Workflow
1. **Validate inputs**:
   - Theme family must be one of: plum, ocean, dagobah, dathomir, hoth, tatooine
   - Project name must be non-empty
2. **Generate pages**:
   - Each `DesignPage` → `.razor` file (generated via `RazorCodeGenerator`)
   - Filename derived from route: `/customers` → `Customers.razor`
3. **Include design model**:
   - Serialize entire `DesignDocument` as `design/document.json`
   - Enables future re-import, audit, versioning
4. **Prepare templates**:
   - Read embedded template files (csproj, layouts, shared components)
   - Substitute `{{ProjectName}}` and `{{ThemeFamily}}` placeholders
5. **Package as zip**:
   - Create in-memory zip file (System.IO.Compression)
   - Return as `ExportResult` with byte array for download

### Validation
```csharp
var exporter = new ProjectExporter();
var result = exporter.ExportProject(document, "MyApp", "ocean");
// Throws ArgumentException if validation fails
```

### Theme Support
All six theme families:
- `plum` (default, purple)
- `ocean` (blue)
- `dagobah` (green)
- `dathomir` (red)
- `hoth` (light blue)
- `tatooine` (orange)

### Return Value
```csharp
public record ExportResult(
    string ProjectName,
    string ThemeFamily,
    byte[] ZipData,
    DateTime ExportedAt);
```

### V1 Scope Limitations
- ❌ **No Razor code import**: Code generation is one-way only
- ❌ **No custom logic generation**: C# code is boilerplate; business logic must be added post-export
- ❌ **No API/backend setup**: Focus is on UI only
- ⚠️ **Template embedding in progress**: Full zip with all files in phase 5

## 10. How to Use

### Generate Razor Code
```csharp
var generator = new RazorCodeGenerator();
var document = new DesignDocument { Name = "My App", Version = "1.0" };
var page = new DesignPage { Route = "/home", Title = "Home" };

// Add nodes to page.Nodes...

var razorCode = generator.GeneratePageCode(page, document);
File.WriteAllText("Home.razor", razorCode);
```

### Export a Project
```csharp
var exporter = new ProjectExporter();
var result = exporter.ExportProject(document, "MyApp", "ocean");

// Download the zip
File.WriteAllBytes("MyApp.zip", result.ZipData);
```

### Use the Code Panel in Designer
```razor
<DesignerCodePanel 
    CurrentPage="@selectedPage"
    Document="@designDocument"
    OnDocumentChanged="@HandleDocumentChange" />
```

## 12. Lessons Learned

1. **Determinism is hard**: Timestamps, GUIDs, dictionary iteration order all affect output
   - Solution: No-op timestamps in production, injectable for testing

2. **Monaco Integration Scope**: Real Monaco requires bundling and JS interop
   - V1: Simple tabs with textarea; upgrade to Monaco in phase 5

3. **Client-side Export Requirements**:
   - System.IO.Compression must work in WASM (verify early in phase 5)
   - Zip format must be strictly valid (test with standard unzip tools)

4. **Template Embedding Strategy**:
   - Embedded resources avoid drift (single source of truth)
   - Build-time validation: embedded set must match filesystem
