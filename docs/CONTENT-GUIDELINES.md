# Content Guidelines

These guidelines define the default voice for demo pages, showcase screens, and user-facing copy in this repo.

## Core rules

- Use a professional-direct tone.
- Use nl-NL as the default language; include an EN equivalent when the text is cross-team or reusable.
- Keep button labels verb-first: `Opslaan`, `Werkorder aanmaken`, `Filters wissen`.
- Avoid `OK`, `Klik hier`, and other generic labels.
- Keep system text free of exclamation marks.
- Make error messages tell what happened and what to do next.
- Write empty states as an invitation to continue, not an apology.
- Use sentence case for headings, labels, and helper text.
- Use real examples in placeholders instead of repeating the label.
- Use nl-NL date and number formatting in examples and prose.
- Prefer plain abbreviations only when they are common to the audience; expand them the first time when the term may be ambiguous.

## Good and bad examples

### Buttons

| Good | Bad |
| --- | --- |
| `Opslaan` | `OK` |
| `Werkorder aanmaken` | `Klik hier` |
| `Filters wissen` | `Reset alles` |

### Errors

| Good | Bad |
| --- | --- |
| `De opslag is vol. Verwijder een oud bestand en probeer het opnieuw.` | `Fout 500: payload parse failed` |
| `De e-mail is ongeldig. Voer een adres in zoals naam@bedrijf.nl.` | `Invalid email address.` |

### Empty states

| Good | Bad |
| --- | --- |
| `Nog geen werkorders. Maak de eerste werkorder aan om te beginnen.` | `Er zijn geen resultaten.` |
| `Geen filters actief. Voeg een filter toe om kleiner te zoeken.` | `Sorry, niets gevonden.` |

### Placeholders

| Good | Bad |
| --- | --- |
| `Bijv. Werkorder 1042` | `Werkorder` |
| `Zoek op klant, adres of nummer` | `Zoeken` |

## Review checklist

- Button text says what the user can do.
- Status and error messages are actionable.
- Empty states point to the next step.
- Copy is sentence case.
- Examples use nl-NL formatting.
